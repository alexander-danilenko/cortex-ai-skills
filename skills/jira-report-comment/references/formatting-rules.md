# Formatting Rules

- Use standard markdown (headers, bold, tables, bullet lists)
- Keep the total length between 300-800 words
- Avoid code blocks and technical jargon unless necessary to describe an API endpoint, configuration change, or other technical detail that requires showing exact values
- Use present tense ("Verification now checks..." not "Added checking for...")
- Do not include file paths, class names, or method names
- Group related changes logically, not by commit chronology
- Use simple English - the team may include non-native speakers
- Only use common IT terms (API, endpoint, database, deploy, etc.)
- No em dashes or en dashes - use regular hyphens (-) only
- No special quotes - use regular "" or '' only
- No semicolons or parenthetical asides
- Write like a human teammate posting a quick update, not a formal document
