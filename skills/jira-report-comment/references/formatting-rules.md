# Formatting Rules

- Summary: 1 to 2 sentences, roughly 25 words.
- Root Cause (bug fixes only): max 2 bullets, max 25 words per bullet.
- Verify and Deploy: max 3 bullets per section, max 15 words per bullet.
- Total comment length: 60 to 100 words for feature work, up to 150 words when Root Cause is present.
- Common IT terms only (API, endpoint, database, deploy). No file paths, class names, or method names.
- No code blocks unless an exact value (env var, endpoint path, flag name) is essential.
- Present tense framed as outcomes ("Verification now checks the NPI Registry"), not activities ("Added NpiRegistryService").
- No em or en dashes - use regular hyphens (-) only.
- No special quotes - use regular "" or '' only.
- No semicolons or parenthetical asides.
- Sound like a teammate posting a quick update, not a formal report.
